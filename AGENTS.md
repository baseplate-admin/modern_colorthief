# AGENTS.md

Instructions for AI agents working on the modern_colorthief project.

## Project Overview

High-performance color palette extraction from images. Rust core with bindings for Python, Node.js, WASM, JVM, and Ruby. Manual median-cut algorithm with rayon parallelism.

## Architecture

- **`crates/core/`** — Manual median-cut algorithm with rayon. Accepts raw RGBA buffers. No external image decoding. No `color-thief` crate.
- **`crates/python-bindings/`** — PyO3 bindings. Pillow for image decoding (transparent in Python layer). Rust receives raw RGBA bytes.
- **`crates/node-bindings/`** — napi-rs bindings. Sharp for image decoding. TypeScript preferred. Bundled with vite.
- **`crates/wasm-bindings/`** — wasm-bindgen. Canvas API for image decoding. WebGPU planned for GPU acceleration.
- **`crates/jvm-bindings/`** — JNI bindings. Java AWT for image decoding. Java 26 target. Android compatible. Kotlin bindings.
- **`crates/ruby-bindings/`** — magnus bindings. `image_processing` gem for image decoding.
- **`crates/*-gpu/`** — GPU-accelerated bindings using Vulkan Compute (cross-vendor: Intel, AMD, NVIDIA). WASM uses WebGPU.

## Key Rules

1. **No deprecated features** — Use current APIs only.
2. **Same API across all bindings** — `getPalette(image, colorCount?, quality?)` and `getColor(image, quality?)`. Image decoding is language-specific.
3. **Shared deps in workspace `Cargo.toml`** — rayon, colorthief-core. Implementation-specific deps in crate folders.
4. **Core is manual median-cut with rayon** — No `color-thief` crate, no `image` crate in bindings.
5. **Package names**: `@modern_colorthief/node`, `@modern_colorthief/wasm`
6. **CI**: Separate workflow per binding. Python CI is maturin-autogenerated (only add test step). Node tested against Node, Deno, Bun. JVM tested with Java 26 + Kotlin. npm publishes use `--provenance` for OIDC attestation.
7. **Tests**: Same test format across all bindings (test_api, test_errors, test_properties, test_edge_cases).
8. **llms.txt**: Per-binding API docs. Index + full. Auto-regenerated on API change via hooks.
9. **GPU**: Vulkan Compute for native (cross-vendor). WebGPU for WASM.
10. **vite** for bundling/type generation in Node bindings.
11. **vite-plugin-dts** for type declaration generation.
12. **uv monorepo** — Root pyproject.toml links to python-bindings.

## Build Commands

```bash
cargo test -p colorthief-core                    # Core
cd crates/python-bindings && maturin develop     # Python
cd crates/node-bindings && npx napi build --platform --release  # Node
cd crates/wasm-bindings && wasm-pack build --release           # WASM
cargo build --release -p modern-colorthief       # JVM
cargo build --release -p ,modern-colorthief      # Ruby
```

## Rust Conventions

- `py.detach(move || { ... })` for Python to release GIL during heavy work
- `#[pyfunction]` with `#[pyo3(signature = (...))]` for defaults
- Errors: `.map_err(PyValueError::new_err(...))`
- Module uses `gil_used = false`
- Release: `opt-level=3`, fat LTO, strip, single codegen unit, panic=abort
- Rayon `par_iter()` for parallel pixel processing

## Python Conventions

- Public API: `get_palette()`, `get_color()` with full docstrings
- Input dispatch via `match`: `str` → path, `bytes`/`BytesIO` → bytes, `PIL.Image` → direct
- Type hints on all functions, `py.typed` marker present
- CLI uses `argparse`, entry point: `modern-colorthief`
